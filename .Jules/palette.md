## 2024-05-15 - [Enhancing Keyboard Navigation and Accessibility in Custom CLI-like UI Components]
**Learning:** Terminal-like interfaces in the web often rely on completely hiding browser default focus styles (`focus:outline-none` or `focus:ring-0`) to achieve a specific CLI aesthetic. When combined with inputs that lack `aria-label`s, this causes critical accessibility regressions. Screen reader users lack context on what the input is for, and keyboard users completely lose track of their focus position on the screen since there is no visual indicator that the input is active.
**Action:** Always provide `aria-label` on inputs missing explicit `<label>` tags. Instead of stripping focus indicators entirely, move the focus ring to the parent container wrapping the input using `focus-within:ring-1` combined with an appropriate theme color (e.g., `#2EED8F` for the main chat, `blue-400` for Jules). Additionally, mark decorative elements like prompt symbols (e.g., `>_` or `❯`) with `aria-hidden="true"` so they are not confusingly announced by screen readers.
